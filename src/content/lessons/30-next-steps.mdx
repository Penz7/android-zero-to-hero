### Kotlin Multiplatform (KMP)

```kotlin
// Kotlin Multiplatform chia sẻ code giữa platforms

// shared/src/commonMain/kotlin
// Code chung cho tất cả platforms
class UserRepository {
    fun getUsers(): List<User> { /* ... */ }
}

// shared/src/androidMain/kotlin
// Code riêng cho Android
actual class Platform actual constructor() {
    actual val name: String = "Android ${Build.VERSION.SDK_INT}"
}

// shared/src/iosMain/kotlin
// Code riêng cho iOS
actual class Platform actual constructor() {
    actual val name: String = "iOS ${UIDevice.currentDevice.systemVersion}"
}

// shared/build.gradle.kts
kotlin {
    androidTarget()
    iosX64()
    iosArm64()
    iosSimulatorArm64()
    
    sourceSets {
        commonMain.dependencies {
            implementation("org.jetbrains.kotlinx:kotlinx-coroutines-core:1.7.3")
            implementation("org.jetbrains.kotlinx:kotlinx-serialization-json:1.6.0")
        }
    }
}
```

### Compose Multiplatform

```kotlin
// Compose Multiplatform - UI code chung cho Android, iOS, Desktop, Web

// commonMain
@Composable
fun App() {
    MaterialTheme {
        var count by remember { mutableStateOf(0) }
        
        Column(
            modifier = Modifier.fillMaxSize(),
            horizontalAlignment = Alignment.CenterHorizontally,
            verticalArrangement = Arrangement.Center
        ) {
            Text("Count: $count")
            Button(onClick = { count++ }) {
                Text("Increment")
            }
        }
    }
}

// Chạy trên Android
fun main() = singleWindowApplication {
    App()
}

// Chạy trên iOS
fun MainViewController() = ComposeUIViewController { App() }

// Chạy trên Desktop
fun main() = application {
    Window(onCloseRequest = ::exitApplication) {
        App()
    }
}
```

### Open Source Contribution

```markdown
Why Contribute to Open Source?
- Build reputation
- Learn from experienced developers
- Improve code quality
- Give back to community

How to Start:
1. Find beginner-friendly issues
   - Labels: "good first issue", "help wanted"
   - Repositories: android/architecture-samples, square/retrofit

2. Start small
   - Fix typos in documentation
   - Add unit tests
   - Improve error messages

3. Follow contribution guidelines
   - Read CONTRIBUTING.md
   - Follow code style
   - Write tests for new features

4. Be professional
   - Respond to feedback
   - Be patient with review process
   - Learn from criticism

Recommended Projects:
- Jetpack Compose samples
- Kotlin libraries
- Android architecture samples
- Material Design components
```
