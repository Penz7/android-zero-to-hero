## Phỏng vấn Android Developer

### Câu hỏi phỏng vấn phổ biến

**1. Kotlin Fundamentals:**
```kotlin
// Q: Sự khác biệt giữa val và var?
// A: val là immutable (không thể gán lại), var là mutable

// Q: Data class là gì? Khi nào dùng?
// A: Data class tự động tạo equals(), hashCode(), copy(), toString().
// Dùng cho DTOs, entities, API responses

// Q: Null safety trong Kotlin?
// A: Nullable types (?), safe call (?.), Elvis (?:), non-null assertion (!!)

// Q: Extension functions?
// A: Thêm hàm mới vào class hiện có mà không cần kế thừa

// Q: Sealed class vs Enum?
// A: Sealed class linh hoạt hơn, mỗi subclass có thể có state khác nhau
```

**2. Coroutines:**
```kotlin
// Q: Dispatchers trong Coroutines?
// A: Main (UI), IO (network/db), Default (CPU-intensive), Unconfined

// Q: StateFlow vs SharedFlow?
// A: StateFlow giữ giá trị hiện tại, SharedFlow có thể có nhiều subscribers

// Q: Cách handle exception trong Coroutines?
// A: try-catch, CoroutineExceptionHandler, SupervisorJob
```

**3. Compose:**
```kotlin
// Q: Recomposition là gì?
// A: Compose redraw UI khi state thay đổi

// Q: remember vs rememberSaveable?
// A: remember giữ qua recomposition, rememberSaveable giữ qua config changes

// Q: State hoisting là gì?
// A: Chuyển state lên component cha, component con trở nên stateless

// Q: LazyColumn vs Column với verticalScroll?
// A: LazyColumn chỉ render visible items, hiệu quả hơn cho danh sách dài
```

**4. Architecture:**
```
// Q: MVVM là gì?
// A: Model-View-ViewModel: tách UI (View) và business logic (ViewModel)

// Q: Repository pattern?
// A: Abstract data sources, cung cấp single source of truth

// Q: Clean Architecture?
// A: Domain (entities, use cases), Data (repository, data sources), Presentation (UI, VM)

// Q: Dependency Injection?
// A: Cung cấp dependencies từ bên ngoài, giúp testable và decoupled
```

### STAR Method cho Behavioral Questions

```markdown
STAR = Situation - Task - Action - Result

Example: "Hãy kể về project bạn làm việc nhóm"

S (Situation): "Trong project cuối khóa, nhóm 5 người làm app quản lý chi tiêu"

T (Task): "Tôi đảm nhận phần Architecture và CI/CD pipeline"

A (Action): 
- Setup MVVM với Clean Architecture
- Implement Hilt DI cho testability  
- Tạo GitHub Actions CI/CD pipeline
- Review PRs và pair programming với teammates

R (Result): 
- App hoàn thành đúng hạn
- 90% code coverage
- Deploy thành công lên Play Store
- Nhóm đạt điểm A+
```

### Technical Questions - Code Challenge

```kotlin
// Q: Viết hàm đảo ngược chuỗi
fun reverseString(s: String): String = s.reversed()

// Q: Tìm số lớn thứ 2 trong list
fun secondLargest(numbers: List<Int>): Int? {
    return numbers.distinct().sortedDescending().getOrNull(1)
}

// Q: Flatten nested list
fun flatten(list: List<Any>): List<Any> = list.flatMap {
    if (it is List<*>) flatten(it.filterNotNull()) else listOf(it)
}

// Q: Implement debounce
fun <T> Flow<T>.debounce(timeoutMillis: Long): Flow<T> = flow {
    coroutineScope {
        val downstream = MutableSharedFlow<T>()
        launch {
            collect { downstream.emit(it) }
        }
        downstream.debounce(timeoutMillis).collect { emit(it) }
    }
}

// Q: Cách xử lý configuration changes?
// A: ViewModel (survive config changes), SavedStateHandle (survive process death)
```

### Design Patterns Questions

```kotlin
// Q: Singleton trong Kotlin?
object Database {
    fun query() { /* ... */ }
}

// Q: Observer pattern?
// A: LiveData, StateFlow, RxJava

// Q: Builder pattern?
class UserBuilder {
    private var name: String = ""
    private var age: Int = 0
    
    fun name(name: String) = apply { this.name = name }
    fun age(age: Int) = apply { this.age = age }
    fun build() = User(name, age)
}

// Q: Strategy pattern?
interface SortStrategy {
    fun sort(list: List<Int>): List<Int>
}

class AscendingSort : SortStrategy {
    override fun sort(list: List<Int>) = list.sorted()
}

class DescendingSort : SortStrategy {
    override fun sort(list: List<Int>) = list.sortedDescending()
}
```
